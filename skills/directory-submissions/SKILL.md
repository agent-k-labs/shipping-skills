# Directory Submissions

Directories are a one-time effort that pays back for months. A well-submitted app on the right directories gets you: organic search traffic, qualified users who are specifically looking for tools, and backlinks that improve your SEO. This skill lists exactly where to submit and what to write.

## Context

Read `product-context.md` if present. Otherwise ask for:
1. Product URL
2. Product category (AI tool / SaaS / browser extension / developer tool / no-code tool / general app)
3. Pricing (free / freemium / paid only)
4. Is the product live now or still in beta/waitlist?

## Process

### 1. Core submission assets (write once, use everywhere)

Before submitting anywhere, prepare these assets — they're used across almost every directory:

- **Name:** [Product name]
- **Tagline:** Under 60 characters. "What it does" not "who you are."
- **Short description (100 chars):** One sentence. Lead with outcome.
- **Medium description (150–300 chars):** Problem + solution + CTA.
- **Long description (up to 500 words):** Full landing page copy compressed.
- **Logo:** 512×512 PNG, white background or transparent, clean
- **Screenshots (3–5):** 1280×800. Show the product doing the thing. No filler.
- **Tags/keywords (up to 10):** What someone would type to find a tool like this
- **Pricing tag:** Free / Freemium / Paid / Free Trial
- **Category:** Pick the most specific category that fits

Prepare all of these before submitting to the first directory. You'll paste them faster.

### 2. Priority tier list

Read [references/directories.md](references/directories.md) for the full tiered directory database (name, URL, category fit, cost). Select from it based on the product's category and stage:

- **Tier 1** (day 1): Product Hunt, Indie Hackers, Show HN, Uneed — every product
- **Tier 2** (week 1): category directories — AI directories only if the product is AI-first; BetaList only if pre-launch
- **Tier 3** (first month): specialized and comparison directories
- **Tier 4**: niche — built-with showcases, marketplace listings, industry-specific lists

Skip directories whose audience doesn't match: a B2B tool on an AI-toy directory earns a worthless backlink and zero users.

### 3. Submission schedule

Do not submit to 30 directories in one day. Spread it out:

```
Day 1: Product Hunt (or schedule launch date), Indie Hackers product page
Day 2: There's An AI For That, Futurepedia (if AI)
Day 3: BetaList or Uneed
Day 4: Show HN, SaaS Hub
Day 5: AlternativeTo, G2
Week 2: Remaining Tier 2 directories
Week 3–4: Tier 3 and niche directories
```

### 4. What to write for each submission

Use the assets from Step 1, but adapt the description to each platform's audience:

- **Product Hunt:** Write the PH description (see `/product-hunt-launch`)
- **AI directories:** Emphasize what AI does in the product, not generic marketing copy
- **B2B directories (G2, Capterra):** Emphasize ROI and integrations
- **Developer directories (HN, Peerlist):** Emphasize the technical angle and how it was built

### 5. After submission: backlink tracking

Most directories link back to your site. Track them:
- Set up Google Search Console (free)
- Add your site to Ahrefs free tier or Moz Link Explorer
- After 2 weeks, check which directories indexed your listing and which drove traffic

## Output

1. **Submission asset kit** — all 8 assets written and ready to copy-paste
2. **Customized directory list** — Tier 1–3 directories ranked by relevance to this product
3. **30-day submission calendar** — day-by-day schedule
4. **Per-directory description variants** — PH, AI-directory, and B2B-directory versions of the description
5. **Tracking note** — what to check in Google Search Console after 2 weeks
